Ch 1. Multi-Archetype Profiles

Define precise configurations for game characters, NPCs, enemies, animals, or specialized melancholic actors used in story-driven narrative arcs.

define asset_profile named "kratos_clone" :
    set asset_archetype HERO
    set visual_render_hex 0xffffff
    set baseline_movement_speed 18

define asset_profile named "crying_villager" :
    set asset_archetype SAD_STORY_ACTOR
    set visual_render_hex 0x64748b

Ch 2. Story Playback Takeover

The core framework supports swapping player controllers at runtime. This dynamically converts a standard background NPC into the active playable asset for custom timeline sequences.

if narrative_checkpoint eq dramatic_flashback :
    execute dynamic_controller_takeover target_entity "crying_villager" tracking_time 30.0_sec

Ch 3. Visual Placement Sync

Coordinates parsed dynamically by the interactive canvas engine automatically serialize straight into the master script manifest using precise 3D spatial notation tags.

place asset named "hero" at coordinates(X: 0.00, Y: 1.10, Z: 0.00)
place asset named "heo" at coordinates(X: 8.54, Y: 1.00, Z: -5.20)

Ch 4. Core Interaction Rules

Calculates combat resolution matrices, environmental interactions, and dialogue execution pipelines based on entity-specific tag variables.

[GAMEPLAY_START]
if hero punch enemy1:
    let enemy health -10

if hero kicks car:
    car gothrough
[GAMEPLAY_END]
